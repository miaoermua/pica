# Pica 社区行为准则、开发规范与使用条款

感谢您关注并使用 **Pica**。为了维护一个健康、专业且可持续发展的开源生态，所有 Pica 程序的使用者、软件源维护者以及贡献者均需遵守本规范。

---

## 1. 开发声明与维护准则
*   **开发溯源与 AI 辅助：** Pica 在制作过程中参考了众多优秀的开源项目，并深度结合了 AI 辅助生成技术(vibe coding)进行代码实现与逻辑优化。
*   **维护者门槛：** 我们欢迎社区贡献。但由于 Pica 结合了特定实践，参与维护者必须具备理解相关引用项目底层原理以及 Pica 核心逻辑的能力。
*   **分支与生态一致性：** 我们非常注重 Pica 的软件质量与技术实现。由于分支导致的随意修改与再分发极易造成社区分裂与后续维护困难，因此我们强烈要求：在创建公共分支或衍生项目前，请务必与核心团队沟通并达成共识，如不遵循社区规范且拒绝沟通，请勿创建相关项目。

## 2. 软件源、程序传播与合规性规范
任何提供 Pica 软件源镜像或分发程序的组织与个人，必须遵守以下红线：

*   **工具与配置分离：** 
    *   Pica 仓库允许打包和分发工具，但在软件包中我们不会携带相关服务提供。
    *   Pica 仅提供技术实现工具，不对用户自行配置的内容承担法律责任。
*   **内容纯净性：** 严禁制作、传播任何携带破坏性代码、恶意篡改系统、窃取隐私或含有后门程序的软件包。
*   **法律红线：** 禁止打包、托管或传播违反分发者或使用者所在地区法律法规的任何软件内容。
*   **发布范围限制：** 禁止擅自传播 Pica 路线图中尚未计划、尚未成熟或处于开发阶段的实验性功能。
*   **真实宣传：** 提倡客观真实，严禁夸大宣传、虚假承诺或任何可能误导用户的表述。

不建议打包或贡献，协议不清晰混乱的应用，软件质量需足够可靠。

## 3. 社区互动与安全防范
*   **尊重与秩序：** 严禁无故攻击、诋毁开发者、贡献者或镜像服务提供方。
*   **技术反馈：** 常规 Bug 或功能建议请通过 GitHub 的 Issues 或 Discussions 提交。
*   **安全漏洞：** 
    *   **公开披露：** 为防止漏洞被恶意利用，严禁在公开渠道披露未修复的漏洞细节。
    *   **快速联系：** 发现任何安全风险，请立即联系：`miaoermua@gmail.com` 或 `i@miaoer.net`。
    *   **维护保障：** 若核心开发者长时间未回复且项目长期未维护，请联系活跃的维护者或协助者进行后续沟通。

---
*本规范旨在保护 Pica 的技术纯粹性与社区安全性。感谢每一位维护开源秩序的伙伴。❤️*
